CalTrust

Adaptive Trust Calibration via Contextual Bandits

The problem: Different users need different types of calibration. A naive user might be overwhelmed by technical uncertainty expressions, while an expert needs precise confidence intervals. One-size-fits-all calibration doesn't work.

The solution: CalTrust uses a LinUCB contextual bandit to select among four calibration interventions based on user context and output characteristics. An XGBoost trust predictor (AUC 0.70) trained on 19,200 simulation trials guides the bandit's decisions.

Why this matters for HCI

Shows that one-size-fits-all uncertainty presentation is suboptimal — different users and contexts require different calibration strategies, making personalisation an HCI design challenge.

Key Insight

The bandit correctly learns "no intervention" for naive users (~46.7% of cases), avoiding unnecessary cognitive load. This is a ceiling effect: naive users already accept ~89% of LLM outputs, and any calibration intervention risks reducing trust rather than building it. The system prioritizes do no harm over maximal intervention.

Tech Stack

Python 3.12 XGBoost LinUCB scikit-learn Streamlit 19.2K trials

Key Takeaways

  1. LinUCB contextual bandit selects among 4 intervention types based on user + output context.
  2. XGBoost trust predictor: AUC 0.70, RMSE 1.04 on 19,200 simulation trials.
  3. Bandit learns "no intervention" for naive users (~46.7%), avoiding unnecessary cognitive load.
