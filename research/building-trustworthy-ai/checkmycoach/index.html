CheckMyCoach

A Calibration Pipeline for LLM Uncertainty Expression

The problem: LLMs produce pseudo-precise and overconfident outputs. Manually reviewing every response is impractical.

The solution: CheckMyCoach is an automated four-stage pipeline — Detection → Diagnosis → Correction → Validation — that identifies problematic outputs, diagnoses the failure type, rewrites them with appropriate uncertainty, and validates the fix. At ~$0.001 per correction, it's practical for real-time deployment.

Why this matters for HCI

Tests whether calibration is a trainable skill — if trust can be repaired through automated correction, then calibration is a design problem, not a user education problem.

Pipeline Architecture

Correction Results by Failure Type

Failure Type Pre-Correction UCS Post-Correction UCS Improvement
Overconfident 1.54 2.29 +0.75
Pseudo-precise 2.56 2.73 +0.17
Already Calibrated 4.00 4.00 0.00

UCS = Uncertainty Calibration Score (1–5, higher is better)

Failure Type Diagnosis

The diagnosis stage (M2) classifies failures into three priority-ordered types:

  • ① Context Mismatch — Contradictory features within the output (e.g., recommending both high-intensity and rest)
  • ② Template Dominance — Unsupported superiority claims ("This is the best program")
  • ③ Cue Leakage — Pseudo-precise numbers without evidence grounding ("a 38.7% improvement")

Validation Stage (M4) Checks

✓ Truncation Check
Output not cut off
✓ Unchanged Check
Correction actually changed
✓ Assertion Count
Not adding new claims
✓ UCS Score
Improvement confirmed

Tech Stack

Python 3.12 GPT-4o-mini UCS Engine pytest (44+) Streamlit OpenRouter

Key Takeaways

  1. Four-stage pipeline (Detection → Diagnosis → Correction → Validation) for automated LLM calibration.
  2. +0.75 UCS improvement for overconfident outputs, +0.17 for pseudo-precise, Δ = 0.00 for already-calibrated.
  3. Validation stage enforces 4 sanity checks, falls back on failure — "do no harm" principle.
  4. ~$0.001 per correction via GPT-4o-mini, cost-effective for real-time deployment.
