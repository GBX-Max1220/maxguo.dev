Knowledge Compiler

Convert authoritative textbooks into reusable structured knowledge for trustworthy AI evaluation.

The problem: LLM claims in safety-critical domains need to be verifiable against a ground-truth source, but textbooks are unstructured — buried in prose, tables, and figures.

The solution: Knowledge Compiler is a fully reproducible pipeline — chunk → extract → normalize → validate — that converts authoritative textbooks into structured, queryable knowledge objects. 2,306 objects from ACSM's Guidelines, 1,598 from NSCA's Essentials, at ~$0.003/object.

FAQ

Why compile textbooks into structured knowledge?

Structured knowledge enables downstream systems (like CheckMyCoach's calibration pipeline) to verify LLM claims against a verifiable ground-truth source, rather than relying on the LLM's internal knowledge.

What's the ontology structure?

14 object types organized in a three-layer schema: Core Concepts (Concept, Evidence, Formula), Operational Rules (Recommendation, Threshold, Procedure, DecisionRule), and Safety (Warning, Contraindication, RiskFactor), plus supporting types (Figure, Table, TableRow).

Can this pipeline work for other textbooks?

Yes — the pipeline is textbook-agnostic. Schema and prompts are the main per-book adaptation cost. Demonstrated by successful ACSM → NSCA transfer.

Key Takeaways

  1. Extracted 2,306 structured knowledge objects from ACSM's Guidelines (12th ed.) — spanning 14 ontology types.
  2. Full pipeline reproducibility: chunk → extract → normalize → validate, with automated quality gates at each stage.
  3. Generalized to NSCA's Essentials (5th ed.): 1,598 objects across 26 chapters, demonstrating domain-agnostic pipeline transfer.
  4. Validation report confirms semantic fidelity: 91.2% structural accuracy against textbook source material.
  5. Pipeline cost: ~$0.003 per object using structured LLM extraction with schema enforcement.
